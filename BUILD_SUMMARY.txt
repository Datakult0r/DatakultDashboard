================================================================================
                         COMPONENT OVERVIEW
================================================================================

StatCard
  Props: number, label, icon?, bgColor?, numberColor?, subtext?
  Usage: Display large metrics with context
  Example: Urgent count with red icon and warning color

SourceTag
  Props: source (string), sourceType? (SourceType)
  Usage: Identify origin of triage item
  Returns: Colored pill badge (80px max)

ScoreChip
  Props: score (number), label (ScoreLabel), className?
  Usage: Show job/item quality score with label
  Features: Mono font, pulsing animation for priority

TriageCard
  Props: item (TriageItem)
  Usage: Display urgent or review items
  Features: Expandable draft reply, copy button, external link

JobCard
  Props: item (TriageItem)
  Usage: Show job opportunities in pipeline
  Features: Salary, location, recruiter info, easy-apply badge

NewsCard
  Props: item (TriageItem)
  Usage: Display news items with thumbnails
  Features: Image on left (80x60px), source badge, external link

ScheduleRow
  Props: item (TriageItem), now? (Date)
  Usage: Show calendar events with time awareness
  Features: Current/soon highlighting, status badges, location

DashboardShell
  Props: initialItems (TriageItem[]), initialStats (TriageStat | null)
  Usage: Main dashboard container (client component)
  Features: Tabs, stats, real-time subscription, responsive layout

To Deploy:
  1. Ensure Supabase project is set up with:
     - triage_items table (with all columns)
     - triage_stats view (with aggregations)
  
  2. Set environment variables:
     NEXT_PUBLIC_SUPABASE_URL
     NEXT_PUBLIC_SUPABASE_ANON_KEY
  
  3. Build: npm run build
  4. Deploy to Vercel/your preferred Next.js host

The dashboard will automatically:
  - Fetch today's data on load
  - Subscribe to real-time updates
  - Refresh stats as data changes
  - Handle all responsive layouts
  - Maintain dark theme throughout
